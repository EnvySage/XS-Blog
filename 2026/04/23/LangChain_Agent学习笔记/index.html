LangChain Agent 学习笔记

学习日期：2026年4月23日
使用模型：通义千问 (Qwen3-Max)

一、Agent 基础概念

Agent（智能体）是 LangChain 中的核心组件，能够让 LLM 自主决策并调用工具完成任务，而不仅仅是生成文本。

1.1 Agent vs 普通 LLM

特性 普通 LLM Agent
工具调用 ❌ ✅
自主决策 ❌ ✅
多步骤推理 ❌ ✅
环境交互 ❌ ✅

三、ReAct 框架

3.1 什么是 ReAct

ReAct = Reasoning + Acting

一种让 Agent 通过「思考 → 行动 → 观察」循环来解决问题的推理框架。

3.2 ReAct 案例：计算 BMI

3.3 ReAct 执行流程

  1. 思考 (Thought)：分析问题，决定下一步行动
  2. 行动 (Action)：调用工具获取信息
  3. 观察 (Observation)：分析工具返回结果
  4. 循环直到得到最终答案

3.4 监控工具调用

四、中间件 (Middleware)

4.1 中间件类型

装饰器 时机 用途
@before_agent Agent 执行前 初始化、日志
@after_agent Agent 执行后 资源清理、记录结果
@before_model 模型调用前 请求预处理
@after_model 模型调用后 响应后处理
@wrap_model_call 模型调用中 监控、拦截
@wrap_tool_call 工具执行中 记录工具参数

5.3 工具描述的重要性

LLM 根据 description 判断何时调用工具，描述应该：

  • 清晰说明工具用途
  • 说明必要的输入参数
  • 用自然语言描述预期行为

七、学习总结

7.1 核心要点

  • Agent = LLM + 工具：让模型具备执行能力
  • ReAct 框架：思考-行动-观察的循环推理
  • 中间件：监控和干预 Agent 执行流程
  • 工具描述：决定 LLM 如何选择和使用工具

7.2 与 LangChain 基础对比

概念 LangChain 基础 Agent
调用方式 链式调用 自主决策
工具使用 手动调用 自动选择
状态管理 手动管理 自动维护
多步骤 预设流程 动态规划

八、后续学习方向

  1. Tool Calling：深入学习 LLM 原生工具调用机制
  2. Plan-and-Execute Agent：规划-执行分离模式
  3. Self-Ask Agent：自我提问推理
  4. Memory Agent：带记忆的长期任务 Agent
  5. 多 Agent 协作：多个 Agent 配合完成任务
